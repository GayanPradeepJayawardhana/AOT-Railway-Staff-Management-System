## Testing Relationships

### User → Role Relationship
**Relationship Method:**
```php
public function role()
{
    return $this->belongsTo(UserRole::class,'role_id');
}
```
**Test Command:**
```php
$user->role;
```
**Expected Output:**
```
id: 2
role_name: Super Administrator
role_slug: super_admin
```

### User → Station Relationship
**Retrieve Specific User:**
```php
$user = App\Models\User::where('nic_number','199912345678')->first();
```
**Test Command:**
```php
$user->station;
```
**Expected Output:**
```
station_code: BRL
station_name: Beruwala
```

### Station → Users Relationship
**Retrieve Specific Station:**
```php
$station = App\Models\Station::where('station_code','BRL')->first();
```
**Test Command:**
```php

```
**Expected Output:**

## Important Table Names Reference

| Model | Table Name | Custom Table Setting |
|-------|------------|---------------------|
| UserRole | user_roles | Default |
| PasswordChangeHistory | password_change_history | `protected $table='password_change_history';` |
| MissingSubmissionsTracking | missing_submissions_tracking | `protected $table='missing_submissions_tracking';` |
| Station | stations | Default |
| User | users | Default |
| Designation | designations | Default |
| MonthlyReport | monthly_reports | Default |

---

## Troubleshooting Table Name Issues

If Laravel cannot automatically guess table names, add explicit table declarations in your models:

```php
// For PasswordChangeHistory Model
protected $table='password_change_history';

// For MissingSubmissionsTracking Model
protected $table='missing_submissions_tracking';
```
